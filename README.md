# 无线网络上下层联动的身份认证实验

## 项目简介 (Project Context)
本实验是《高级计算机网络实验》课程的实验内容。实验通过模拟无线网络环境，实现了一个结合**上层密码学认证**与**底层物理层指纹特征**的跨层联动身份认证系统。

系统核心利用 RSA 挑战-应答（Challenge-Response）机制确保用户身份的不可伪造性，并涵盖了合法接入、未注册拦截以及仿冒攻击检测等实验场景。

## 依赖项 (Dependencies)
本项目基于 Python 3.8+ 开发。核心加密部分依赖 `rsa` 库。

- `rsa`: 处理 RSA 密钥生成、公钥解析及数字签名验证。
- `json`: 用于客户端与服务器的消息序列化。
- `socket`: 实现基于 TCP 协议的网络通信。

**安装依赖：**
```bash
pip install rsa
```

## 文件结构 (File Structure)
- `server.py`: 认证服务器。维护公钥数据库，处理注册与认证请求。
- `client.py`: 交互式客户端。自动演示三个核心实验场景（合法、未注册、仿冒）。
- `test.py`: 自动化单元测试。使用 `unittest` 框架验证协议状态机与逻辑正确性。
- `Wi-Fi data/`: CSV 数据集。包含 6 个发射机的特征数据，用于物理层认证扩展。

## 运行指南 (Running Instructions)

> **注意**：进行认证前必须先启动服务器。

### 1. 启动服务器（server）
```bash
python server.py
```
服务器将监听在 `127.0.0.1:65432`。

### 2. 运行场景模拟 (Client)
```bash
python client.py
```
该脚本将按顺序演示：
- **场景 1 (User_A)**: 合法注册并通过 RSA 认证。
- **场景 2 (User_B)**: 未注册直接认证，被服务器拦截。
- **场景 3 (User_C)**: 攻击者冒用 A 的 ID 发起认证，因签名不匹配失败。

### 3. 运行自动化测试 (Test)
```bash
python test.py
```
可选项

## 实验逻辑说明
1. **注册阶段**: 客户端发送 `user_id` 和公钥 PEM 字符串，服务器存入字典。
2. **认证阶段**: 
   - 客户端发送认证请求。
   - 服务器返回 32 字节随机数挑战（Challenge）。
   - 客户端使用私钥对挑战进行 SHA-1 签名。
   - 服务器使用对应公钥验证签名，验证通过后允许进入物理层认证。
